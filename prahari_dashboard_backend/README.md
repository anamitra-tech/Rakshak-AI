# Prahari Backend

FastAPI backend. Every endpoint works right now against placeholder
services (`app/services/*_placeholder.py`) — fixed, obviously-fake
responses in the correct shape, so the frontend is fully testable
before the AI/ML developer's real classifier/entity-extraction/graph
files are dropped in. Each placeholder file documents the exact
function signature the real file needs to match.

## Setup

```bash
cd backend
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env            # fill in real keys later, works blank too
```

## Run

```bash
uvicorn app.main:app --reload --port 8000
```

Check it's alive:
```bash
curl http://localhost:8000/api/health
```
```json
{"status": "ok"}
```

Interactive API docs (auto-generated by FastAPI, useful for testing
without curl): http://localhost:8000/docs

## Endpoints

| Method | Path | Module |
|---|---|---|
| POST | `/api/analyze` | Unified classifier (Citizen Fraud Shield + digital-arrest-style scams) |
| POST | `/api/extract_entities` | Entity extraction |
| GET | `/api/network/graph` | Fraud Network |
| GET | `/api/network/clusters` | Fraud Network |
| GET | `/api/geo/complaints` | Geospatial |
| GET | `/api/geo/districts/stats` | Geospatial |
| GET | `/api/health` | — |

Full request/response shapes are in `../prahari-api-contract.md` (or
`app/models/schemas.py` for the source of truth).

## What's placeholder vs final right now

| Piece | Status |
|---|---|
| Classifier intelligence (`/api/analyze`) | **Placeholder** — fixed fake result. Real file must match `classify(text, mode) -> dict` in `app/services/classifier_placeholder.py` |
| Entity extraction (`/api/extract_entities`) | **Placeholder** — fixed fake result. Real file must match `extract_entities(text) -> dict` in `app/services/entity_extraction_placeholder.py` |
| Fraud network graph/clusters | **Placeholder** — fixed fake nodes/edges/clusters. Real file must match `get_graph()`/`get_clusters()` in `app/services/graph_placeholder.py` |
| Geospatial complaints/stats | Synthetic demo data only (`geo_service.py`) — not part of the AI/ML handoff |
| Schemas, routes, CORS, error handling, auth | **Final** |

## Handing off to the AI/ML developer

Point them at the three `*_placeholder.py` files in `app/services/` —
each has a top-of-file comment with the exact function signature and
return shape their real file needs to match so it can be dropped in
with a one-line import change in the corresponding `app/api/*.py` route
(see the `# --- SWAP THIS IMPORT ---` comment in each route file).

Everything else (`main.py`, `api/*.py`, `models/schemas.py`, CORS,
error handling) is the plumbing — don't need to touch it to improve
the intelligence layer.

## Wiring to the frontend

In your Vite `vite.config.js`, add a proxy so the frontend can call
`/api/...` directly without hardcoding `localhost:8000` or hitting
CORS issues:

```js
export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': { target: 'http://localhost:8000', changeOrigin: true },
    },
  },
});
```
